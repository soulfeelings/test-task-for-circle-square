# The Last of Guss

Браузерная игра, в которой выжившие соревнуются, кто больше натапает по мутировавшему гусю.

## Структура монорепозитория

- `backend/` — Fastify + TypeORM сервис, PostgreSQL в качестве хранилища.
- `frontend/` — клиент на React, Vite и React Router.

## Требования

- Node.js 20+
- PostgreSQL 14+

## Настройка окружения

1. Создайте базу данных PostgreSQL.
2. Скопируйте `backend/.env.example` в `backend/.env` и заполните переменные.
3. Установите зависимости и выполните миграции:

```bash
npm install --workspaces
(cd backend && npm run build)
# примените миграцию TypeORM, например через CLI: npm run typeorm migration:run
```

## Запуск

### Backend

```bash
cd backend
npm run build
node dist/index.js
```

Сервер стартует на порту `PORT` из `.env` (по умолчанию 3000).

### Frontend

```bash
cd frontend
npm run dev
```

Vite поднимет dev-сервер на `http://localhost:5173` и будет проксировать запросы `/api` на backend.

## Основные возможности

- Регистрация/авторизация по username + password (первый вход создаёт пользователя).
- Роль определяется автоматически: `admin` для администратора, `Nikita`/`Никита` — особая роль, чьи тапы не дают очков.
- Администратор создаёт раунды. Длительность раунда и задержка старта настраиваются через переменные окружения.
- Выжившие тапают гуся в активном раунде, каждый 11-й тап приносит 10 очков.
- Реализована защита от гонок при подсчёте очков за счёт транзакций и блокировок записей.
- Страница раунда показывает статус, счёт, лидеров и индивидуальную статистику игрока.
