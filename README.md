# The Last of Guss

Тестовый проект, включающий минималистичный бэкенд и фронтенд для браузерной игры «The Last of Guss». Игроки соревнуются, кто быстрее натапает мутировавшего гуся, получая очки в течение ограниченного раунда.

> ⚠️ В репозитории отсутствуют сторонние зависимости из npm из-за ограничений окружения. Весь код написан на TypeScript и работает поверх стандартных возможностей Node.js и браузера.

## Стек

- **Backend**: Node.js (ES modules, TypeScript strict), собственный HTTP-сервер, файловое хранилище в формате JSON
- **Frontend**: TypeScript, ванильный DOM, собственный роутер (hash-based)

## Быстрый старт

### Переменные окружения

Создайте файл `.env` в каталоге `backend` (можно скопировать из `.env.example`). Доступны параметры:

```env
ROUND_DURATION=60        # длительность раунда в минутах
COOLDOWN_DURATION=30     # задержка перед стартом в минутах
JWT_SECRET=supersecret   # секрет для токенов
PORT=3000                # порт сервера
DATA_FILE=./data/storage.json
```

### Сборка и запуск бэкенда

```bash
cd backend
npm run build
npm run start
```

Сервер поднимается на `http://localhost:3000` и хранит данные в JSON-файле. Для чистого запуска удалите файл, указанный в `DATA_FILE`.

### Сборка фронтенда

```bash
cd frontend
npm run build
```

После сборки откройте `frontend/public/index.html` (используется `dist/main.js`). Для работы требуется запущенный бэкенд на `http://localhost:3000`.

## Основные возможности

### Роли пользователей

- **admin** — может создавать раунды
- **survivor** — обычный игрок
- **nikita** — тапы успешны, но в статистике всегда показывают ноль очков

Роль определяется по имени пользователя при регистрации (admin, Никита/Nikita, остальные).

### API (без внешних библиотек)

| Метод | Маршрут | Описание |
|-------|---------|----------|
| `POST` | `/auth/login` | Логин/регистрация по username/password, возвращает токен |
| `GET` | `/rounds` | Список раундов для авторизованного пользователя |
| `POST` | `/rounds` | Создание раунда (только admin) |
| `GET` | `/rounds/:id` | Детали раунда, включая личную статистику |
| `POST` | `/rounds/:id/tap` | Тап по гусю в активном раунде |

Токен передаётся через заголовок `Authorization: Bearer <token>`.

### Правила начисления очков

- Каждый тап — 1 очко
- Каждый 11-й тап добавляет 10 очков (но не для Никиты)
- Никита видит и получает нули в статистике, однако запросы обрабатываются

## Ограничения и допущения

- Из-за отсутствия Postgres используется файловое хранилище с примитивной блокировкой. Для продакшн-окружения рекомендуется заменить на настоящую БД.
- Фронтенд не использует React/Vite из-за недоступности npm-пакетов. Реализован собственный лёгкий роутер и работа с DOM.
